莫队学习笔记

发表于 2023-04-08 分类于 学习笔记
WIP

前言

莫队是一种用于解决静态区间查询问题的有力工具．

约定

序列长 nnn ，询问数 qqq 和值域大小 VVV 同级．

若未加说明，下文中 lll，rrr 均指代莫队中维护的两个指针．

普通莫队

如果已知一个区间 [l,r][l, r][l,r] 的信息，可以在小代价内转移得到 [l±1,r±1][l \pm 1, r \pm 1][l±1,r±1] 的信息，那么我们可以使用普通莫队算法在 O(nn)O(n \sqrt{n})O(nn​) 的转移次数内得到所有询问区间的信息．

算法思想

对于这样的问题，我们很容易有一个朴素的想法：维护两个指针 lll、rrr，按照询问顺序移动 lll、rrr 并维护区间 [l,r][l, r][l,r] 的信息．但这样 lll 和 rrr 的移动次数都达到了 O(n2)O(n^2)O(n2)，不可接受．

如果指针的移动有单调性就好了！于是我们将询问离线，按照左端点排序，再移动指针．这样我们就将 lll 的移动次数降到了 O(n)O(n)O(n)．但不幸的是，排序之后，右端点就失去了单调性，rrr 的移动次数可达 O(n2)O(n^2)O(n2)．

按照左端点排序会使右端点失去单调性，反之亦然．于是我们想到根号平衡，可以让左端点损失一定的单调性，换来右端点一定的单调性，这样总移动次数就可以被平衡到一个可以接受的值．

于是我们按照块长 n\sqrt{n}n​ 将序列分块，以左端点所在的块作为第一关键字，右端点作为第二关键字排序，再按照这个顺序移动指针，移动次数为 O(nn)O(n \sqrt{n})O(nn​)．

复杂度分析

设块长为 BBB．

对于排序后两个相邻的询问，由于按左端点所在块的编号排了序，lll 的移动次数不会超过 O(B)O(B)O(B) 次，则 lll 的总移动次数为 O(nB)O(nB)O(nB)．

而对于一个块中的所有询问，右端点单调，rrr 的移动次数为 O(n)O(n)O(n)，总共有 O(n/B)O(n / B)O(n/B) 个块，那么 rrr 的总移动次数为 O(n2/B)O(n^2 / B)O(n2/B)．

总移动次数为 O(nB+n2/B)O(nB + n^2 / B)O(nB+n2/B)，B=nB = \sqrt{n}B=n​ 时最优，为 O(nn)O(n \sqrt{n})O(nn​)．

例题

例 1. SPOJ DQUERY D-query

静态区间数颜色，考虑使用莫队解决．

莫队维护数 iii 在当前区间内出现的次数 cic_ici​．插入 xxx 时，若 cx=0c_x = 0cx​=0，说明 xxx 在当前区间第一次出现，维护的答案自增，删除时同理．总时间复杂度 O(nn)O(n \sqrt{n})O(nn​)．

例 2. 洛谷 1494 [国家集训队] 小 Z 的袜子

记 cic_ici​ 表示 iii 在当前区间出现的次数．对于一个询问区间 [l,r][l, r][l,r]，答案为：

∑i(ci2)(r−l+12)=1(r−l+1)(r−l)∑ici(ci−1)\sum_i \frac{\binom{c_i}{2}}{\binom{r - l + 1}{2}} = \frac{1}{(r - l + 1)(r - l)} \sum_i c_i (c_i - 1) i∑​(2r−l+1​)(2ci​​)​=(r−l+1)(r−l)1​i∑​ci​(ci​−1)

莫队维护 ccc 和 ∑ici(ci−1)\sum\limits_i c_i (c_i - 1)i∑​ci​(ci​−1)，显然可以 O(1)O(1)O(1) 移动指针，总时间复杂度 O(nn)O(n \sqrt{n})O(nn​)．

例 3. 洛谷 4396 [AHOI2013] 作业

维护 cic_ici​ 和 did_idi​，分别表示区间中 iii 出现的次数和 iii 是否出现过．did_idi​ 的维护类似于例 1．

询问是一个区间求和．为了支持区间求和操作，可以选择直接使用 BIT 维护 ccc 和 ddd，但是这样移动指针的代价就是 O(log⁡n)O(\log n)O(logn)，时间复杂度 O(nnlog⁡n)O(n \sqrt{n} \log n)O(nn​logn)，难以通过本题．

考察莫队的算法流程，我们发现它会在内层数据结构上产生 O(nn)O(n \sqrt{n})O(nn​) 次修改和 O(n)O(n)O(n) 次查询，而这显然是不平衡的．如果我们能够使用 O(1)O(1)O(1) 修改，O(n)O(\sqrt{n})O(n​) 查询的数据结构替换 BIT，那么就可以将时间复杂度平衡到 O(nn)O(n \sqrt{n})O(nn​)．

那么我们采用分块来维护 ccc 和 ddd，容易做到 O(1)O(1)O(1) 移动指针，O(n)O(\sqrt{n})O(n​) 查询区间和．总时间复杂度 O(nn)O(n \sqrt{n})O(nn​)．

回滚莫队

在普通莫队中，当移动指针使得当前维护的区间长度增加时，我们称这次移动为插入，反之则称这次移动为删除．

当插入和删除操作中有一个难以实现时，可以采用回滚莫队算法在 O(nn)O(n \sqrt{n})O(nn​) 的转移次数内得到所有询问区间的信息．

算法思想

以难以实现删除操作的情况为例．

我们发现，即使是难以实现删除操作的信息，也可以快速实现撤销到某个特定点的操作．那么我们能不能基于插入和撤销两个操作，得到所有询问区间的信息呢？

考察普通莫队算法中给询问排序的作用，发现其本质是将询问挂在了左端点所在的块上，并保证挂在一个块上的所有询问右端点的单调性，从而保证时间复杂度．

考虑左端点在一个块中的所有询问，右端点已经单调增了，不需要对 rrr 实施删除操作．但左端点是乱序的，为了规避对 lll 的删除操作，处理每个询问时，我们可以将 lll 撤销到当前块的右端点上，再进行移动．

遍历所有块．设当前块的右端点为 br\text{br}br，将 lll 初始化为 br\text{br}br，rrr 初始化为 br−1\text{br} - 1br−1，并且清空维护的所有信息，然后按上述操作处理左端点在当前块中的所有询问．

注意这样的算法无法处理左右端点在同一个块内的询问，这些询问暴力做即可．

难以实现插入操作的情况类似，不同之处如下：

  • 给挂在一个块上的询问排序时，按照右端点降序排序．
  • 遍历所有块时，设当前块的左端点为 bl\text{bl}bl，将 lll 初始化为 bl\text{bl}bl，rrr 初始化为 nnn．

复杂度分析

同样以难以实现删除操作的情况为例．

设块长为 BBB．

对于每个询问，设询问左端点为 ql\text{ql}ql，ql\text{ql}ql 所在块的右端点为 br\text{br}br，lll 都会从 br\text{br}br 移动到 ql\text{ql}ql，移动次数为 O(B)O(B)O(B)．总移动次数为 O(nB)O(nB)O(nB)．

rrr 的移动次数分析和普通莫队一致．

总移动次数为 O(nB+n2/B)O(nB + n^2 / B)O(nB+n2/B)，B=nB = \sqrt{n}B=n​ 时最优，为 O(nn)O(n \sqrt{n})O(nn​)．

难以实现插入操作的情况类似．
